       - 第1行：动作标签 + 发送者（"${actionLabel} by @${sender}"）
       - 第2行：上下文信息（区分 issue/PR："Issue #${issueNumber}: ${issueTitle}" 或 "PR #${issueNumber}: ${issueTitle}"）

         * 验证 payload.comment 存在且为对象
         * 验证 payload.issue 存在且为对象
       - 动作过滤：
         * 检查 action 是否在 SUPPORTED_ACTIONS 中
         * 不支持的动作记录警告并返回 { processed: false, message: "Unsupported issue_comment action: ${action}", event: 'issue_comment', data: { action, issueNumber } }
       - 数据提取（使用占位符）：
         * commentBody：payload.comment.body（字符串且非空）或 '[No comment]'
         * issueTitle：payload.issue.title（字符串且非空）或 '[No title]'
         * issueNumber：payload.issue.number 或 null
         * commentUrl：payload.comment.html_url 或 null
         * senderLogin：payload.sender.login 或 'unknown'
         * isPr：!!payload.issue.pull_request（检查是否存在此字段以区分 PR 评论）
       - 调用 formatCommentMessage 格式化消息
       - 返回结构：
         {
           processed: true,
           message,
           event: 'issue_comment',
           data: {
             action,
             issueNumber,
             issueTitle,
             commentAuthor: senderLogin,
             isPr
           }
         }

    6. 添加完整的 JSDoc 注释：
       - 模块说明
       - 常量说明
       - 函数说明（@param, @returns, @example）
       - @throws 标注错误条件

    7. 确保遵循现有代码风格：
       - 使用 async/await
       - 使用 const/let（非 var）
       - 使用 === 而非 ==
